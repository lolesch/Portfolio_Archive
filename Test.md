We tested system scalability against a high demand of customization. Our desktop-first e-commerce is build around a 'single-product' approach, using variables and -modes to control the outcome. After a systemic proof of concept we developed an atomic design system and refined it’s design after gathering insights from usability testing.

# Results

- customization blueprint that could be expanded/scaled
- editor as core piece
- full e-commerce flow
- dynamic slots -> custom and extendable content

The product editor is the core of the user experience. Using variables and -modes, we implemented a blueprint for customization that can be adapted to product requirements.
Users are guided by information hierarchy through the shopping progress - all prototyped and applicable to our design principles.

## Learnings
- [ ] what are the takeaways from this learning experience?
- [ ] list all the learnings from variable usage and usability testing

- no support for smart animating variable mode changes.
- issues with 'scroll to' setups when using instance variants
- agile usability testing worked best for us due to fast iteration cycles
- do not rush wireframing
	- this was a flaw from the project setup, where we needed to get to a testable state ASAP
- design-principles and -system are very powerful when applied early on and consistently
